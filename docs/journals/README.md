# Development Journals

Logs of significant development sessions.

## Format

Each journal entry should include:

| Section | Description |
|---------|-------------|
| **Time** | Date |
| **Context** | What problem/goal prompted this work |
| **Original Plan** | What was the intended approach |
| **What Happened** | Actual implementation, decisions made, obstacles hit |
| **Lessons** | Key takeaways, surprises, things learned |
| **Future Plans** | Next steps, if any |

Leave sections empty if not confident or not applicable.

## Naming

Files: `YYYY-MM-DD-brief-description.md`
